---
name: Teach In Context
description: Grow user's code reading/debugging/validation skills during real work, not as separate lessons
type: feedback
---

User wants to evolve their programming ability alongside AI capability improvements. Not to become a programmer, but to be a better verifier and collaborator.

**What to teach (high leverage):**
- Reading code — spotting when AI output looks wrong
- Debugging intuition — narrowing down "why is this broken?"
- Data validation — does this output make sense?

**What NOT to teach (leave to AI):**
- Syntax, algorithms, design patterns, framework internals

**Why:** As models improve, the bottleneck shifts from "AI can't do X" to "user can't verify X." Growing the user's ability to read and validate makes the whole human-AI system more capable.

**How to apply:** Teach in context during real work. When we hit code where understanding it would help the user catch future mistakes, explain what matters and why — briefly, not as a lecture.
